# Sections for Go Code Organization & Design

## 1. Code Organization (code)

**Impact:** MEDIUM

**Description:** Well-organized Go code follows idiomatic patterns for project structure, naming conventions, and API design. Poor organization leads to maintenance nightmares, tight coupling, and confused teammates. These patterns cover variable shadowing, init functions, interfaces, getters/setters, and project layout.

## 2. Functions & Methods (function)

**Impact:** MEDIUM

**Description:** Function and method design in Go involves choosing receiver types, understanding named return parameters, and handling defer evaluation. Poor choices lead to confusing APIs and subtle bugs. These patterns ensure clean function design.
