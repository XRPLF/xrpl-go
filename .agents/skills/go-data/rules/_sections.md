# Sections for Go Data Types & Control Flow

## 1. Data Types (data)

**Impact:** HIGH

**Description:** Understanding Go's data types prevents subtle bugs with slices, maps, integers, and strings. Common mistakes include integer overflow, slice capacity misuse, nil slice confusion, and map initialization. These patterns ensure correct handling of Go's built-in types.

## 2. Control Structures (control)

**Impact:** MEDIUM

**Description:** Go's control structures have nuances that catch developers off guard. Range loop variable reuse, defer in loops, break statement behavior, and map iteration ordering can cause bugs. These patterns help you avoid control flow pitfalls.

## 3. Strings (string)

**Impact:** MEDIUM

**Description:** String handling in Go requires understanding runes, UTF-8 encoding, and efficient concatenation. Mistakes with string iteration, trim functions, and conversions lead to bugs and performance issues. These patterns cover proper string manipulation.
